# 프로젝트 아키텍처 및 유지보수 가이드

이 문서는 Members-JS 프로젝트의 구조와 향후 유지보수를 위한 기술 가이드라인을 제공합니다.

## 1. 현재 기술 스택 (Current Tech Stack)
* **HTML5**: 웹 표준 및 시맨틱 태그 활용, 접근성 및 SEO 고려
* **CSS3**: 반응형 웹 디자인(미디어 쿼리), 커스텀 스타일링 (`reset.css` 및 `style.css` 분리)
* **Vanilla JavaScript**: DOM 조작 및 UI 인터랙션 (의존성 최소화)

## 2. 폴더 및 파일 구조
```text
Members-JS/
├── index.html           # 메인 홈페이지
├── about.html           # 병원 및 원장님 소개
├── eye.html             # 눈 성형 안내
├── nose.html            # 코 성형 안내
├── lifting.html         # 리프팅 시술 안내
├── botox.html           # 보톡스/필러 시술 안내
├── transport.html       # 오시는 길 및 진료 시간
├── css/
│   ├── reset.css        # 브라우저 기본 스타일 초기화
│   └── style.css        # 메인 및 공통 스타일
├── js/
│   └── javascript.js    # 공통 인터랙션 스크립트 (네비게이션, 모달 등)
├── assets/              # 주요 이미지 리소스 (업스케일링된 고화질 에셋)
├── icons/               # 파비콘 및 각종 아이콘 리소스
└── docs/                # 기획, 구조, 협업 규칙 등의 문서화 폴더
```

## 3. 유지보수 가이드 (Maintenance Guide)

### 3.1. 이미지 교체 및 추가
* **용량 최적화**: 웹사이트 로딩 속도 저하를 막기 위해 이미지는 가급적 WebP 포맷을 사용하거나 용량을 압축하여 업로드합니다.
* **SEO (대체 텍스트)**: 새로운 `<img>` 태그를 추가할 때는 반드시 `alt` 속성을 자세히 기입하여 검색 엔진 노출 확률을 높입니다.
* **업스케일링**: 화질이 낮은 이미지는 AI 업스케일링 툴을 거쳐서 반영하는 것을 권장합니다.

### 3.2. 반응형 디자인(Responsive Web)
* 모바일 환경(스마트폰, 태블릿)에서의 가독성을 최우선으로 고려합니다.
* 새로운 UI 요소를 추가할 경우 `style.css` 하단의 미디어 쿼리(Media Query) 섹션을 확인하고 모바일 뷰가 깨지지 않는지 점검합니다.

## 4. 향후 고도화 계획 (Future Migration)
본 프로젝트는 추후 **Spring Framework** 환경으로 마이그레이션 및 고도화될 예정입니다.

* **동적 웹 구성**: 정적(Static) HTML 파일을 Spring Boot 기반의 타임리프(Thymeleaf)나 JSP, 또는 별도의 프론트엔드(React/Vue) 프레임워크와 API 서버(Spring)로 분리할 예정입니다.
* **관리자 페이지**: 병원 공지사항이나 이벤트 팝업 등을 원장님(관리자)이 직접 업로드할 수 있도록 관리자(Admin) 기능을 구축합니다.
* **DB 연동**: 방문자 통계나 간단한 문의 내역 등을 데이터베이스(MySQL 등)와 연동하여 관리합니다.
